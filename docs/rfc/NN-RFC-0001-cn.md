# NN-RFC-0001

# 语义共识协议

Version 1

## 摘要

语义共识协议定义了 nyanya LLM Agent 之间如何交换、比较和融合部分解释，而不将其降级为 token 级别的对齐。本协议规定了 nyanya LLM 分布式推理流水线中的消息格式、路由、融合规则、异常处理及稳态约束。

## 状态

协议规范。取代 Draft 0。

参考文献：[NN-TR-2026-001](https://github.com/nya-lab/NN-TR-2026-001)。

## 1. 术语

`Field`（场）

共享语义环境。所有 Agent 向 Field 贡献并从中读取。Field 不是数据库——它是通过持续信号交换维护的分布式状态。

`Signal`（信号）

Agent 发出的有界解释。信号是协议中唯一的通信单元。每个信号携带来源、置信度和张力。

`Tension`（张力）

兼容但不完整的解释之间的冲突。张力不是错误——它是达成共识的原材料。协议保留张力而非过早消解。

`Merge`（融合）

将张力消解为协调状态的行为。一次有效的融合应当保留未消解的张力，声明有损压缩，并允许结果状态被挑战。

`Drift`（漂移）

由重复摘要或不稳定路由导致的语义损失。漂移量化为信号与其经过协议栈重新编码后形式之间的语义距离。

`OSU`（有机语义单元）

Organic Semantic Unit。跨节点传输的最小标准化载体。定义见 §2。

`OPU`（有机处理单元）

Organic Processing Unit。CBNF 织物中的异构认知 Agent。定义见 NN-TR-2026-001 §5.1。

`Route`（路由）

通过 OPU 集群的有向路径。路由是动态的，可因应张力、漂移或饱和而重新配置。

## 2. 协议目标

本协议优化以下目标：

- Agent 间的语义一致性；
- 可追溯的分歧；
- 低语义漂移；
- 可逆融合；
- 简洁的状态传输。

## 3. 非目标

本协议不定义真理。

本协议不声称生物学合理性。

本协议不替代模型推理。

本协议不规定具体的共识算法。它定义消息格式和规则；底层共识机制为实现定义。

## 4. OSU 消息格式

协议中的每条信号均编码为 OSU（有机语义单元）。OSU 是 OPU 外部输出和跨节点传输的最小标准化载体。

### 4.1 编码结构

```
osu {
  id:        string       // 唯一信号标识符
  source:    agent_id     // 来源 OPU
  c:         semantic     // 核心语义内容（§4.2）
  omega:     scalar       // 局部置信权重 [0, 1]
  tau:       ms           // 生成延迟
  rho:       scalar       // 观点冲突系数 [0, 1]
  eta:       scalar       // 信息增量系数 [0, 1]
  provenance: ref[]       // 上游 OSU 引用
  ttl:       uint         // 剩余跳数
}
```

### 4.2 核心语义内容

`c` 字段承载有效载荷。可包含：

- 局部结论；
- 反例；
- 推理经验；
- 视觉联想；
- 逻辑疑点；
- 风险预警；
- 不完整的推导片段。

`c` 字段对路由层不透明。其内部结构由消费方 OPU 定义。

### 4.3 字段语义

- `omega` —— 置信度。值越高表示局部信念越强。协议使用 omega 进行加权投票，但不将高 omega 视为真值。
- `tau` —— 延迟。OPU 生成此信号所花费的墙上时钟时间。NLB 用于分类慢推理片段。
- `rho` —— 冲突度。指示此信号与当前共识的差异程度。高 rho 信号优先进入 ACI 审查。
- `eta` —— 新颖度。衡量此信号相对于已有信号贡献了多少新信息。低 eta 触发 BES 饱和检测。
- `provenance` —— 可追溯性。每条信号必须引用其上游 OSU。由此实现可逆融合和漂移度量。
- `ttl` —— 跳数限制。每次路由跳转递减。ttl=0 的信号被丢弃。防止无限路由循环。

## 5. 融合规则

### 5.1 有效性条件

一次融合在以下条件满足时有效：

- 输入信号保留来源追溯；
- 未消解张力被继续传递；
- 有损压缩被显式声明；
- 结果状态可被后续信号挑战。

### 5.2 融合流程

```
merge(signals: OSU[]) -> OSU:
  1. 按来源链对信号分组
  2. 对每条链：
     a. 计算加权共识：Σ(omega_i * c_i) / Σ(omega_i)
     b. 残余张力 = max(rho_i) 跨输入
     c. 若残余张力 > T_threshold：
          将输出标记为 "pending-inspection"
          入队等待 ACI 验证
  3. 组合输出 OSU：
     c = 聚合共识
     omega = 输入 omega 的调和平均
     rho = 残余张力
     eta = 相对上一共识的信息增益
     provenance = 输入信号 id 列表
```

### 5.3 ACI 审查

对抗性共识审查（ACI）机制对高置信度共识簇进行多层逆向验证。当融合输出超过安全阈值时：

```
A = max(Contradiction(z), Blindspot(z), Fragility(z))
```

其中 z 为被审查的共识簇。若 A 超过安全阈值，结论标记为待定，路由至 NLB 异步缓冲池，等待差异化 OPU 补充信号后进行二次收敛。

## 6. 路由协议

### 6.1 动态路由

Flow 模块按以下规则将信号路由通过 OPU 集群：

```
route(signal, cluster) -> OPU[]:
  1. 按语义类别对信号分类
  2. 选择满足以下条件的 OPU：
     - s_i（语义先验）匹配信号领域
     - r_i（推理能力）满足任务复杂度
     - a_i（可用注意力）超过最低阈值
  3. 负载均衡：分配信号以避免热点饱和
  4. 注入多样性：为低先验、高 D 的 OPU 保留 20% 容量
```

### 6.2 RES 守卫

递归调度退化（RES）守卫监测路由重复度：

```
RES = R_repeat / (R_novel + epsilon)
```

分级干预：

- **轻度（RES > threshold_1）**：轮换活跃 OPU 集合，引入具有高认知多样性 D 的空闲单元。
- **中度（RES > threshold_2）**：重构 Split 阶段的任务分解，打破既有的固定路由路径。
- **重度（RES > threshold_3）**：强制终止循环调度链路，输出当前共识并附不确定性标注，停止非增量计算消耗。

## 7. 异常处理

### 7.1 LAXI 监测

大规模自适应经验交互（LAXI）接口持续监测五类异常：

- 认知疲劳累积；
- 语义特征漂移；
- 重复调度冗余；
- 高熵语义堆积；
- 共识收敛阻塞。

复合异常指标：

```
Lambda_i = gamma_1 * F_i + gamma_2 * S_i + gamma_3 * A_i + gamma_4 * R_i
```

其中：
- F_i：单元疲劳度；
- S_i：语义漂移幅度；
- A_i：异常输出占比；
- R_i：循环调度频次。

### 7.2 分级处置

当 Lambda_i 超过阈值时，节点进入代谢处置状态：

1. 单元权重衰减；
2. 局部上下文清除；
3. 异常信号标记滞留；
4. 路由链路切换；
5. 单元临时休眠；
6. 回退至低风险推理路径。

无法立即整合的碎片语义归类为 DSR（延迟语义残留），在延迟批次中回收处理。

## 8. 稳态约束

### 8.1 CCL 均热

认知潜能均衡回路（CCL）监测单元热度：

```
T_i = w_e * E_i + w_v * V_i + w_h * H_i
```

当 T_i 超过全局安全阈值时，回路自动下调该单元的全局投票权重，阻断不受约束的单边认知潜能扩张所导致的全局观点偏移。

### 8.2 协调扩展律

集群性能 P 受以下约束：

```
P = A * N^alpha * K^beta * (1 - sigma)^gamma * D^mu
```

协议必须在最优区间内运行：
- sigma（冲突密度）：适中，避免崩塌（sigma -> 0）和饱和（sigma 过高）；
- D（认知多样性密度）：充足，防止 ESL 回声语义循环；
- N（活跃 OPU 数量）：仅在伴随 D 成比例增长时增加。

### 8.3 BES 饱和

协议监测基础执行饱和度：

```
BES = Repeat(OSU) / (Novel(OSU) + epsilon)
```

当 BES 超过阈值时，MCP 协议下调对应单元的任务配额，LAXI 同步回收冗余语义残留。

## 9. 实现说明

### 9.1 默认参数

| 参数 | 默认值 | 说明 |
|---|---|---|
| T_threshold | 0.3 | ACI 审查的张力阈值 |
| RES threshold_1 | 0.4 | 轻度退化触发 |
| RES threshold_2 | 0.7 | 中度退化触发 |
| RES threshold_3 | 1.2 | 重度退化触发 |
| w_e | 0.4 | CCL 情绪激活权重 |
| w_v | 0.3 | CCL 置信度权重 |
| w_h | 0.3 | CCL 敌意度权重 |
| delta | 500 ms | NLB 延迟阈值 |
| theta_1 | 0.3 | BEW 复杂度分流阈值 |

### 9.2 最小可行实现

符合规范的实现必须支持：

- OSU 消息编解码（§4）；
- 带张力保留的融合（§5）；
- 带多样性注入的动态路由（§6.1）；
- RES 守卫监测（§6.2）；
- LAXI 异常检测（§7.1）。

以下为可选：

- ACI 审查（§5.3）；
- CCL 均热（§8.1）；
- BES 饱和检测（§8.3）。
